## 🔧 الخطوة 1: إعداد GitHub

### 1.1 إنشاء حساب GitHub (إذا لم يكن لديك)

```
1. اذهب إلى: https://github.com
2. اضغط: Sign up
3. أدخل بريدك الإلكتروني
4. اختر كلمة مرور قوية
5. أكمل التحقق
```

### 1.2 إنشاء مستودع جديد

```
1. اضغط: + (أيقونة الإضافة)
2. اختر: New repository
3. أدخل اسم المستودع: quran-bot
4. اختر: Public (عام)
5. اضغط: Create repository
```

### 1.3 نسخ ملفات البوت إلى GitHub

#### الطريقة أ: استخدام Git (من سطر الأوامر)

```bash
# 1. انتقل إلى مجلد البوت
cd /home/ubuntu/quran_bot

# 2. هيّئ مستودع Git محلي
git init
git add .
git commit -m "Initial commit: Quran Bot"

# 3. أضف المستودع البعيد
git remote add origin https://github.com/YOUR_USERNAME/quran-bot.git

# 4. ادفع الملفات إلى GitHub
git branch -M main
git push -u origin main
```

#### الطريقة ب: رفع يدوي (من الويب)

```
1. اذهب إلى مستودعك على GitHub
2. اضغط: Add file
3. اختر: Upload files
4. اسحب الملفات أو اختر من جهازك
5. أضف رسالة commit
6. اضغط: Commit changes
```

### 1.4 التحقق من الملفات على GitHub

تأكد من وجود هذه الملفات في المستودع:

```
quran-bot/
├── bot_final.py          ✅ ملف البوت الرئيسي
├── requirements.txt      ✅ المكتبات المطلوبة
├── Dockerfile            ✅ ملف Docker
├── .env.example          ✅ ملف البيئة النموذجي
├── README.md             ✅ ملف التوثيق
└── .gitignore            ✅ ملف التجاهل
```

## ⚠️ حل المشاكل الشائعة

### المشكلة 1: "Build failed"

**الحل:**
```
1. تحقق من ملف requirements.txt
2. تأكد من أن جميع المكتبات مكتوبة بشكل صحيح
3. اضغط: Manual Deploy
4. جرب مرة أخرى
```

### المشكلة 2: "Service crashed"

**الحل:**
```
1. عرض السجلات
2. ابحث عن الخطأ
3. صحح الخطأ في الكود
4. ادفع التحديث إلى GitHub
5. Render سيعيد النشر تلقائياً
```

### المشكلة 3: البوت لا يرسل الآيات

**الحل:**
```
1. تحقق من متغيرات البيئة
2. تأكد من التوكن صحيح
3. تأكد من معرف القناة صحيح
4. عرض السجلات للأخطاء
```

### المشكلة 4: "Unauthorized" أو "Forbidden"

**الحل:**
```
1. تأكد من أن البوت مسؤول في القناة
2. تأكد من أن البوت لديه صلاحية "نشر الرسائل"
3. أعد إضافة البوت كمسؤول
4. أعد النشر على Render
```
