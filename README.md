> # بوت آيات القرآن الكريم - مشروع تلغرام

بوت تلغرام متقدم لإرسال آيات قصيرة من القرآن الكريم بشكل دوري، مع دعم الأوامر التفاعلية، وإدارة المستخدمين، وضمان الاستقرار والموثوقية.

---

## 🌟 الميزات الرئيسية

- **إرسال دوري**: إرسال آية عشوائية كل 15 دقيقة إلى قناة أو مستخدمين.
- **آيات عشوائية وفورية**: الحصول على آية عشوائية فوراً باستخدام أمر `/verse`.
- **اختيار السور والآيات**: عرض آيات من سورة معينة باستخدام أمر `/surah`.
- **بحث متقدم**: البحث عن آيات تحتوي على كلمات محددة باستخدام أمر `/search`.
- **تحكم المستخدم**: إمكانية إيقاف واستئناف الإرسال الدوري (`/pause`, `/resume`).
- **إدارة التفضيلات**: لوحة إعدادات لتخصيص تجربة المستخدم (قيد التطوير).
- **مفضلة الآيات**: حفظ الآيات المفضلة للرجوع إليها لاحقاً (قيد التطوير).
- **استقرار عالي**: تصميم متقدم لمعالجة الأخطاء، إعادة المحاولة، و `Circuit Breaker`.
- **توثيق شامل**: سجلات مفصلة لتتبع أداء البوت والأخطاء.

---

## 🚀 بدء التشغيل

### 1. المتطلبات الأساسية

- Python 3.10+
- حساب تلغرام
- `pip` لإدارة الحزم

### 2. إعداد البوت

1.  **إنشاء بوت جديد**: تواصل مع `@BotFather` على تلغرام لإنشاء بوت جديد والحصول على **التوكن** الخاص به.
2.  **استنساخ المشروع**:

    ```bash
    git clone https://github.com/your-username/quran-telegram-bot.git
    cd quran-telegram-bot
    ```

3.  **تثبيت المكتبات**:

    ```bash
    pip install -r requirements.txt
    ```

4.  **إعداد متغيرات البيئة**:

    -   انسخ ملف `.env.example` إلى `.env`.
    -   املأ المتغيرات المطلوبة:

        ```env
        # توكن البوت من BotFather
        TELEGRAM_TOKEN=YOUR_BOT_TOKEN_HERE

        # (اختياري) معرف القناة للإرسال العام
        CHANNEL_ID=@your_channel_name
        ```

### 3. تشغيل البوت

```bash
python3 quran_bot.py
```

يفضل تشغيل البوت في بيئة مستقرة مثل `screen` أو `tmux` لضمان استمرارية التشغيل:

```bash
screen -S quran-bot
python3 quran_bot.py
# اضغط Ctrl+A ثم D للخروج من الجلسة مع إبقائها تعمل
```

## 🛠️ الهيكل البرمجي

-   `quran_bot.py`: الملف الرئيسي للبوت، يحتوي على منطق الأوامر والجدولة.
-   `advanced_features.py`: وحدات الميزات المتقدمة مثل لوحات المفاتيح والإحصائيات.
-   `error_handling.py`: فئات إدارة الأخطاء، إعادة المحاولة، و `Circuit Breaker`.
-   `test_quran_bot.py`: اختبارات الوحدة لضمان جودة الكود.
-   `.env`: ملف متغيرات البيئة (يجب عدم رفعه إلى `git`).
-   `requirements.txt`: قائمة المكتبات المطلوبة.

## 📚 الأوامر المتاحة

| الأمر | الوصف |
| --- | --- |
| `/start` | بدء التفاعل مع البوت وعرض رسالة الترحيب. |
| `/verse` | الحصول على آية عشوائية فوراً. |
| `/surah <رقم>` | عرض الآيات الأولى من سورة معينة. |
| `/search <كلمة>` | البحث عن آيات تحتوي على كلمة محددة. |
| `/pause` | إيقاف استلام الآيات الدورية مؤقتاً. |
| `/resume` | استئناف استلام الآيات الدورية. |
| `/help` | عرض قائمة المساعدة وجميع الأوامر. |

## API 🌐 المستخدم

تم الاعتماد على [**Quran Cloud API**](https://alquran.cloud/api) كمصدر أساسي للآيات القرآنية، وذلك للميزات التالية:

-   **مجاني ومفتوح**: لا يتطلب مفتاح API.
-   **سريع وموثوق**: استجابة سريعة وخوادم مستقرة.
-   **شامل**: يدعم الترجمات، الصوت، والبحث.

## 📝 التوثيق الفني

### معالجة الأخطاء

-   **إعادة المحاولة (Retry)**: يتم إعادة محاولة طلبات API الفاشلة تلقائياً باستخدام `requests.Session` مع استراتيجية `Retry`.
-   **قاطع الدائرة (Circuit Breaker)**: لمنع إغراق API بالطلبات الفاشلة، يتم فتح "الدائرة" مؤقتاً بعد عدد معين من الأخطاء المتتالية.

### الجدولة

-   تم استخدام مكتبة `apscheduler` لجدولة إرسال الآيات الدورية كل 15 دقيقة بشكل موثوق.

### إدارة المستخدمين

-   يتم تخزين تفضيلات كل مستخدم (مثل حالة الإرسال) في ملف `user_preferences.json` لضمان استمرارية الإعدادات بين جلسات التشغيل.
